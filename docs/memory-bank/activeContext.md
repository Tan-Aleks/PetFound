# Active Context

## Current Status
MVP перешел из стадии инфраструктурной подготовки в рабочую стадию: аутентификация, реальные данные Supabase, создание объявлений с фото и базовый чат уже реализованы.

## Recent Changes
- **Данные**: Главная и поиск переведены с моков на реальные запросы к Supabase.
- **Аутентификация**: Подключены `NextAuth` и `Supabase Auth` для регистрации/входа.
- **Создание объявления**: Реализована запись в `pets` и загрузка изображений в bucket `pet-photos`.
- **Чат**: Добавлена страница `/chat/[petId]` с чтением и отправкой сообщений из таблицы `messages`.
- **UI/UX**: Внедрено переключение светлой/темной темы и обновлены токены дизайна.
- **Аудит Supabase**: Выявлено, что строгие RLS-политики на `auth.uid()` несовместимы с текущими browser-запросами через anon-клиент без синхронизации Supabase-сессии.
- **Server writes**: Запись объявлений, сообщений, отметка прочтения и загрузка фото переведены на Next.js route handlers с проверкой `NextAuth`-сессии и server-side Supabase client.
- **RLS hardening**: В схеме зафиксирована модель server-only записи, включен `FORCE ROW LEVEL SECURITY` для основных таблиц и удалены политики прямой browser-записи в bucket `pet-photos`.

## Next Steps
1. Вынести server-side чтение чувствительных данных и убрать оставшиеся прямые browser-запросы там, где это нужно для приватности.
2. Реализовать личный кабинет пользователя и управление своими объявлениями.
3. Улучшить UX чата после стабилизации безопасного слоя доступа к данным.
4. Добавить закрытый контур для применения миграций Supabase и проверки production-конфигурации bucket/policies.
