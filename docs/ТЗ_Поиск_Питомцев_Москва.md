# Техническое задание: Платформа поиска пропавших питомцев "ПетПоиск Москва"

## 1. Общее описание проекта

**Цель проекта:** Создание веб-платформы для поиска пропавших домашних питомцев и размещения объявлений о найденных животных в Москве.

**Целевая аудитория:** Владельцы домашних животных, неравнодушные граждане Москвы.

**Географический охват:** Москва и Московская область.

## 2. Функциональные требования

### 2.1 Основной функционал

#### Типы объявлений:
- **"Пропал питомец"** - размещение владельцами
- **"Найден питомец"** - размещение нашедшими

#### Поддерживаемые животные:
- Собаки
- Кошки  
- Мелкие домашние животные (хомяки, попугаи, кролики, морские свинки, шиншиллы и т.д.)

### 2.2 Система регистрации и аутентификации
- **Обязательная регистрация** для размещения объявлений
- **Верификация через телефон и email**
- Технология: NextAuth.js
- Поля регистрации:
  - Имя и фамилия
  - Номер телефона (обязательно)
  - Email (обязательно)
  - Район проживания

### 2.3 Система поиска

#### Расширенный поиск по параметрам:
- Район Москвы
- Тип животного (собака/кошка/мелкие)
- Порода
- Размер (мелкий/средний/крупный)
- Окрас
- Пол
- Дата пропажи/находки
- Особые приметы

#### AI-поиск по изображению:
- Загрузка фото для поиска визуально похожих животных
- Сравнение с базой фотографий в объявлениях
- Показ результатов по степени схожести

#### Интеграция с внешними источниками:
- **Автоматический мониторинг других сайтов** объявлений о животных
- **Парсинг популярных площадок:** Авито, VK группы, Telegram каналы, специализированные сайты
- **AI-анализ внешних объявлений** на соответствие характеристикам искомых питомцев
- **Кросс-платформенные уведомления:** если на другом сайте появилось похожее объявление
- **Обратная связь:** уведомление пользователей других платформ о совпадениях на нашем сайте

### 2.4 Система уведомлений
- **Телефонные звонки** при критических совпадениях
- **SMS-уведомления** о потенциальных находках
- **Email-уведомления** с подробной информацией
- Настройки частоты уведомлений в личном кабинете

### 2.5 Модерация контента
**Гибридная система:**
- Автоматическая проверка на спам и дубликаты
- Фильтрация неподходящего контента
- Ручная модерация подозрительных объявлений
- Система жалоб пользователей

### 2.6 Дополнительные функции

#### Интеграция с социальными сетями:
- Автоматический репост в VK, Telegram
- Кнопки "Поделиться" для расширения охвата

#### Внутренний чат:
- Защищенная переписка между владельцем и нашедшим
- Обмен дополнительными фото и информацией
- История переписки

## 3. Технические требования

### 3.1 Технологический стек
- **Фронтенд:** Next.js (TypeScript), Tailwind CSS, shadcn/ui
- **Бэкенд:** Supabase (PostgreSQL)
- **Аутентификация:** NextAuth.js
- **Хостинг:** Vercel или GitHub Pages
- **Пакетный менеджер:** bun

### 3.2 Архитектура
- **SPA (Single Page Application)**
- **Responsive дизайн** для мобильных устройств
- **PWA возможности** для установки как приложение

### 3.3 База данных (Supabase/PostgreSQL)

#### Основные таблицы:
- `users` - пользователи
- `pets` - объявления о питомцев
- `pet_photos` - фотографии питомцев
- `messages` - внутренние сообщения
- `notifications` - уведомления
- `districts` - районы Москвы
- `external_sources` - внешние источники данных
- `external_pets` - объявления с внешних сайтов
- `cross_matches` - совпадения между платформами
