# Active Context

## Current Focus
- Следующий этап после сохранения `cross_matches`: автоматически создавать `match_found` уведомления и вывести ссылку на внешний матч в UI уведомлений без дублей.
- После согласования плана начинаю следующий этап: сохранить AI-совпадения внешних объявлений в `cross_matches`, чтобы подготовить основу для автоматических уведомлений.
- Добавление AI-сопоставления с внешними объявлениями: поиск по фото должен проверять не только локальные `pets`, но и `external_pets` из партнерских источников.
- Анализ AI-поиска по фото: нужно проверить, сопоставляет ли нейросеть локальные объявления с внешними источниками, и при необходимости добавить этот сценарий.
- Завершение cleanup auth-flow после выноса регистрации в отдельный API: удалить legacy-ветку `register` из `NextAuth Credentials` и синхронизировать документацию.
- Исправлена проблема с регистрацией и входом. Удалён функционал волонтёров.
- Улучшение формы профиля: выбор района проживания переведён на выпадающий список с фиксированными районами Москвы.
- Улучшена навигация личного кабинета: на страницу профиля возвращён верхний Header.
- Устранение серверных ошибок в личном кабинете: защищённые API переведены на server-role Supabase client.
- Устранение остаточных ошибок загрузки данных: публичные server-side маршруты и fallback профиля в auth переведены на более устойчивую схему.
- Регистрация переведена на отдельный API-маршрут вместо `NextAuth Credentials`, чтобы исключить ложный `401` на callback при создании аккаунта.

- Для новых AI-совпадений автоматически создаются `match_found` уведомления без дублей; dropdown уведомлений показывает ссылки на внешний матч и сайт-источник.

- AI-поиск по фото расширен: теперь он ищет совпадения не только среди внутренних объявлений, но и среди `external_pets` с выводом отдельного блока внешних результатов.
- Удалена legacy-ветка `register` из `lib/auth.ts`; `NextAuth Credentials` оставлен только для сценария входа.
- Удалены все файлы, связанные с волонтёрами (страница, API, документация)
- Удалена таблица volunteers из schema.sql
- Удалены типы волонтёров из database.types.ts
- Обновлена документация (ТЗ, README, Memory Bank)
- Добавлены переменные NEXTAUTH_SECRET и SUPABASE_SERVICE_ROLE_KEY в .env.local
- В личном кабинете поле района проживания заменено на выпадающий список с `MOSCOW_DISTRICTS`.
- На странице личного кабинета восстановлен общий верхний Header для быстрого перехода по разделам.
- Защищённые route handlers для профиля, уведомлений и сообщений переведены с `getSupabasePublicServer()` на `getSupabaseServer()`.
- Публичные server-side маршруты объявлений и внешних источников также переведены на `getSupabaseServer()`, а login-flow умеет восстанавливать отсутствующий профиль из `user_metadata`.
- Добавлен `POST /api/auth/register`; страница регистрации теперь сначала создает пользователя через API, а затем выполняет вход отдельным шагом.

## Recent Decisions
- Каноническим источником high-level архитектуры считается `docs/README.md`.
- Каноническим источником прогресса проекта считается `memory_bank/projectbrief.md`.
- Функционал волонтёров удалён, так как не требовался по основному ТЗ.
- Для выбора района в пользовательских формах используется единый справочник `lib/moscow-districts.ts`.
- Для авторизованных server-side API используется service-role клиент Supabase, чтобы маршруты не зависели от публичного anon key.
- Все server-side обращения к Supabase в Next.js routes/pages должны быть независимы от `NEXT_PUBLIC_SUPABASE_ANON_KEY`.
- Регистрация пользователя выполняется отдельным route handler, а `NextAuth Credentials` используется только для входа.
