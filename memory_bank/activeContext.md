## Recent Decisions
- Для `message_received` уведомлений dropdown должен вести сразу в нужный чат по объявлению, а сервер должен создавать такое уведомление в момент отправки нового сообщения.
- Переходы по ссылкам внутри `match_found` уведомлений должны сразу закрывать dropdown и по возможности автоматически снимать unread-state, чтобы пользователь не выполнял лишние действия после навигации.
- AI по внешним совпадениям не выступает как универсальный ассистент: он отвечает только по данным конкретного `match_found` и только на темы источника, локации, контактов и сведений о питомце.
- AI-чат должен визуально жить в разделе сообщений, но быть реализован как отдельный системный сценарий, а не как обычный пользователь в таблице `messages`.
- В элементах `NotificationDropdown` не используются вложенные интерактивные элементы; действия по уведомлению и переходы к связанным сущностям должны быть разнесены на самостоятельные контролы.
- Каноническим источником high-level архитектуры считается `docs/README.md`.
- Каноническим источником прогресса проекта считается `memory_bank/projectbrief.md`.
- Функционал волонтёров удалён, так как не требовался по основному ТЗ.
- Для выбора района в пользовательских формах используется единый справочник `lib/moscow-districts.ts`.
- Для авторизованных server-side API используется service-role клиент Supabase, чтобы маршруты не зависели от публичного anon key.
- Все server-side обращения к Supabase в Next.js routes/pages должны быть независимы от `NEXT_PUBLIC_SUPABASE_ANON_KEY`.
- Регистрация пользователя выполняется отдельным route handler, а `NextAuth Credentials` используется только для входа.
