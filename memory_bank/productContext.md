# Product Context

## Why the project exists
Поиск пропавшего питомца в Москве сейчас разрознен между соцсетями, маркетплейсами и локальными чатами. Проект объединяет ключевые действия в одном интерфейсе: публикацию объявления, поиск и прямую связь между владельцем и нашедшим.

## User Problems
- Объявления о животных быстро теряются в лентах и чатах.
- Поиск по разрозненным площадкам занимает слишком много времени.
- Владельцу и нашедшему нужен быстрый и безопасный канал связи.
- У пользователя нет единого пространства для управления своими объявлениями и откликами.

## Primary Users
- Владелец пропавшего питомца.
- Человек, нашедший животное.
- Волонтер, помогающий локально по районам.

## Product Priorities
1. Быстрое размещение и поиск объявлений.
2. Надежная и безопасная доставка данных через server-side API.
3. Понятный UX для мобильных и desktop пользователей.
4. Подготовка архитектуры к AI-поиску, уведомлениям и внешним источникам.
